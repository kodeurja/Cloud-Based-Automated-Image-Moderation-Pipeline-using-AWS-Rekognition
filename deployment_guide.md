# 🚀 Deployment Guide: Automated Visual Governance & Analytics Dashboard

This guide provides a step-by-step walkthrough to deploy the complete "Governance-as-a-Service" pipeline to your AWS account, as outlined in the project abstract.

## 🏗️ Architecture Overview
1. **Ingestion**: User uploads image to `RawImageBucket`.
2. **Processing**: S3 triggers `VisionAgentLambda`.
3. **Intelligence**: Lambda calls **Amazon Rekognition** (`DetectModerationLabels` & `DetectLabels`).
4. **Persistence**: Structured CSVs are stored in `AnalyticsBucket` under `/moderation-results` and `/label-results`.
5. **Analytics**: **Amazon Athena** queries the S3 Data Lake.
6. **Visualization**: **Amazon QuickSight** displays real-time safety trends.

---

## Step 1: Deploy Infrastructure (CDK)
The core infrastructure is defined using AWS CDK in `infrastructure/cdk_app.py`.

1. **Initialize & Install**:
   ```bash
   npm install -g aws-cdk
   pip install aws-cdk-lib constructs boto3
   ```
2. **Deploy**:
   ```bash
   cd infrastructure
   cdk synth
   cdk deploy
   ```
   *This creates the S3 buckets, IAM roles/policies, and the Lambda function with the necessary S3 triggers.*

---

## Step 2: Configure the Data Layer (Athena)
Once the infrastructure is live, you must tell Athena how to read the CSV files generated by the Lambda.

1. Open the [Amazon Athena Console](https://console.aws.amazon.com/athena/).
2. Run the DDL scripts found in `infrastructure/athena_queries.sql`:
   * **Script 1**: Create `visual_governance_db` database.
   * **Script 2**: Create `moderation_results` table (pointing to your analytics bucket location).
   * **Script 3**: Create `label_results` table.

---

## Step 3: Business Intelligence (QuickSight)
Transform your raw data into a visual dashboard.

1. Open [Amazon QuickSight](https://quicksight.aws.amazon.com/).
2. **Create Dataset**: Choose **Athena** and select the tables created in Step 2.
3. **Build Visuals**:
   * **Pie Chart**: `image_status` (Safe vs. Flagged).
   * **Bar Chart**: `label_name` (Top Violation Categories).
   * **Line Chart**: `timestamp` (Upload trends over time).
   * **KPIs**: Average confidence scores.
   *Refer to `infrastructure/quicksight_setup.md` for detailed mapping instructions.*

---

## Step 4: End-to-End Testing
1. **Upload**: Drop a sample image (e.g., a "safe" landscape or a "test" moderation image like a picture of a weapon) into the `RawImageBucket`.
2. **Verify Lambda**: Check CloudWatch Logs for the `VisionAgentLambda` to see the "Processing image..." messages.
3. **Verify S3 Analytics**: Check the `AnalyticsBucket` for new CSV files appearing in the datetime-prefixed folders.
4. **Refresh Dashboard**: Open QuickSight and verify the data points reflect your test upload.

---

## 🔐 Security & Cost Considerations
* **Serverless**: No EC2 instances to manage; you only pay per image processed.
* **Least Privilege**: The IAM policy is restricted to specific S3 buckets and necessary Rekognition APIs.
* **Data Retention**: Buckets are configured with `RemovalPolicy.DESTROY` for development; update this to `RETAIN` for production environments.
